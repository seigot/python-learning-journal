### Introduction
Hi everyone.
Today, I'm going to talk about my project, the Weekend Library Closing Time Finder.  

First of all, I will introduce my project background.
This project aims to help students efficiently identify libraries that match their weekend schedules.  
Libraries in this system include both public and university libraries.  
In real-world situations, library hours vary between Saturday and Sunday, and some university libraries have access restrictions, such as ID requirements.  
For example, public libraries like the Sunnyvale Public Library are generally open to everyone, while some university libraries may require an ID for access.

### Problem Statement
Some libraries remain open as late as 7:00 PM, allowing students to plan their study sessions flexibly.  
However, studying with too much concentration can cause students to lose track of time and miss their next scheduled activities.  
The key solution of this system is to find libraries whose closing time is closest to but not later than the user’s preferred time.  
This allows students to stay focused on studying while still managing their schedule effectively.  

### Solution(Class)  

Now, I will explain how the system solves this problem.  
The system consists of three main components.  
First, it manages library data using a base Library class, with PublicLibrary and UniversityLibrary defined through inheritance.  
Second, it searches for the optimal closing time based on user input.  
Third, it displays up to three recommended libraries to the user.  

### Solution(Process)  

This diagram shows the overall process of the system.  
There are three main steps.  
First, all library data is loaded into memory before user input.  
Second, the system waits for the user to enter a preferred day and closing time.  
Third, based on the user's input, the system displays up to three recommended libraries.  

From a data structure and algorithm perspective, merge sort is used to sort library closing times, with a time complexity of O(n log n).  
Binary search is used to find the closest closing time, with a time complexity of O(log n), which is very efficient.  
In addition, a queue is used to display the results in FIFO order.  

### Code説明（1）
Next, I will briefly explain how the code works.  
The first class is the Library class.  
- the Library class has get_opening_time/get_closing_time methods to get the library's opening and closing hours.  
- The Library class is the base class, and PublicLibrary and UniversityLibrary inherit from it.  
- The Library class also has a closing_minutes method, which converts closing time into minutes so the system can sort and compare times numerically.
- The UniversityLibrary class has a get_access_condition method, which generates a readable access message, such as whether visitors are allowed or an ID is required.

The second class is LibraryFinderSystem.  
This class finds the best matching libraries by sorting and searching the data.  
- The load_libraries method loads and prepares the data.  
- The merge_sort_libraries method sorts libraries using recursion with  time complexity O(n log n).  
- The binary_search_closest method finds the best match efficiently with time complexity O(log n).  
- After binary search finds the closest valid index, the system collects up to three recommendations from the sorted list.  
- I also use a hash table to store sorted library data for Saturday and Sunday, so the system can access the correct dataset quickly.  
- The find_recommendations method uses the binary search result and collects up to three nearby libraries from the sorted list.  
- The run method controls the main flow of the program, including user input, validation, searching, and displaying results.  

The final class is LibraryResultQueue.  
- The queue is implemented using a linked list, which supports efficient FIFO operations.  
- This class manages and displays results using a queue.  
- The enqueue and dequeue methods manage the data,  
- and the display_results method outputs the results in order.  
- Internally, the queue is built on a linked list. Each node stores one library, and the linked list supports append and pop_left operations for FIFO processing.  
- The utility functions support the system by converting time strings into minutes and validating day and time inputs.
