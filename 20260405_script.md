### Introduction （約1分）
Hi everyone. Today, I'm going to explain about my midterm project "the Weekend Library Closing Time Finder".  

This project aims to help students efficiently identify libraries that match their weekend schedule.  
The system includes both public and university libraries.
In real-world situations, library hours vary between Saturday and Sunday, and some university libraries have access restrictions, such as ID requirements.

### Problem Statement （約1分）  
Some libraries remain open as late as 7:00 PM, allowing students to plan their study sessions flexibly.  
However, studying with too much concentration can cause students to lose track of time and miss their next scheduled activities.  
The key solution of this system is to find libraries whose closing time is closest to but not later than the user’s preferred time.  
This allows students to stay focused and managing their time effectively.

### solutions (class diagram) （約1分）
Now, let me explain how I solve this problem with the class diagram.
The system consists of three main classes.  
The first class is for the Library. The base Library class defines PublicLibrary and UniversityLibrary using inheritance.  
The second class is searching for the optimal closing time.  
The system manages predefined library opening and closing time data, and searches for the best match based on user input.  
The third class is displaying the results. The class aims to show the 3 search results to the user.  

### solutions (activity diagram) （約1分）
This diagram shows the overall process of the system. There are three main steps.
First, all library data is loaded into memory before user input.
Second, the system waits for the user to enter a preferred day and closing time.
Third, based on the user's input, the system displays up to three recommended libraries.

From a data structure and algorithm perspective, 
merge sort is used to sort library closing times, with a time complexity of O(n log n).
Binary search is used to find the closest closing time, with a time complexity of O(log n), which is very efficient.
In addition, a queue is used to display the results in FIFO order.

### Code Demo (Part1.) (2min)
Next, I will briefly explain how the code works.

The first class is the Library class.
This is the base class, and PublicLibrary and UniversityLibrary inherit from it.

The second class is LibraryFinderSystem.
This class finds the best matching libraries by sorting and searching the data.
 - The load_libraries method loads and prepares the data.
 - The merge_sort_libraries method sorts libraries using recursion.
 - The binary_search_closest method finds the best match efficiently.

The final class is LibraryResultQueue.
This class manages and displays results using a queue.
enqueue() and dequeue() method: is for queueing the search result libraries respectively.
 - The enqueue and dequeue methods manage the data in FIFO order.
 - and the display_results method outputs the results in order.
